## 一、核心定位与整体架构
- **产品定位**：轻量级桌面应用，聚焦「MD简历 → PDF简历」的高效转换，主打「实时预览」「便捷编辑」「自定义样式」三大核心体验
- **界面架构**：左右分栏布局，左侧MD编辑区、右侧PDF渲染预览区（支持分栏宽度手动调整，默认2:3）；顶部可增加操作栏（打开侧边栏、模板设置、导出等）

## 二、核心功能需求拆解
### 1. 左侧MD编辑器功能
| 功能分类       | 具体需求                                                                 |
|----------------|--------------------------------------------------------------------------|
| 基础编辑       | 支持MD文本增删改、撤销/重做、实时保存（避免内容丢失）、快捷键 |
| 语法高亮       | 对MD核心语法（标题H1-H6、列表、粗体/斜体、链接、代码块等）进行语法高亮，区分不同文本类型 |
| 文本块拖拽     | 按标题层级（H1/H2为主，适配简历模块如「个人信息」「工作经历」）划分独立文本块；支持拖拽文本块调整顺序，调整后MD源码自动更新，右侧PDF实时同步 |
| 辅助编辑（可选） | 提供简历MD模板预设（基础结构）、MD语法快捷按钮（一键插入标题/列表/链接）、拼写检查 |

### 2. 右侧PDF实时渲染与导出
| 功能分类       | 具体需求                                                                 |
|----------------|--------------------------------------------------------------------------|
| 实时渲染       | 左侧MD内容修改后，右侧PDF预览无感知延迟更新（≤1秒），保留MD原格式（列表、链接、标题层级） |
| CSS模板定制    | ① 内置多款简历PDF模板（简约/商务/学术风格）；② 支持自定义CSS（编写/导入CSS文件），控制字体、颜色、间距、布局；③ 模板管理（保存/删除/切换） |
| 分页控制       | ① 自动分页：根据内容长度+页边距自动拆分页面；② 手动分页：支持MD中插入分页标记（如`<!-- pagebreak -->`），渲染时强制分页 |
| 页边距设置     | 支持自定义PDF上下左右页边距（单位：毫米/像素），实时预览调整效果，支持恢复默认值 |
| PDF导出        | ① 导出「可编辑PDF」（保留文本层，非图片化PDF，可被Acrobat等工具编辑文本）；② 自定义导出路径/文件名；③ 导出时保留MD中的链接、格式，兼容主流PDF阅读器 |

| 功能分类       | 具体需求                                                                 |
|----------------|--------------------------------------------------------------------------|

## 三、最佳技术栈方案建议

### 核心技术栈

| 模块 | 推荐方案 | 理由 |
|------|----------|------|
| 桌面框架 | **Tauri v2** | 仍然是最佳选择，轻量 |
| 前端框架 | **Svelte 5** 或 **Vue 3** | Svelte 编译后体积最小 |
| 构建工具 | **Vite** | 不变 |

### 编辑器：改用 CodeMirror 6 + 自定义块拖拽层

**放弃 bytemd，改用 CodeMirror 6 作为底层编辑器。**

理由：
- CodeMirror 6 架构现代，性能优异，对 Markdown 语法高亮有原生支持
- 可以基于 AST 解析（用 `unified`/`remark` 将 Markdown 解析为 heading 分组），在编辑器上层叠加一个**块拖拽 UI 层**
- Milkdown 基于 ProseMirror 和 Remark，提供了 ProseMirror 基础之上的广泛自定义能力，但需要大量开发投入和手动 UI 构建。

**推荐实现方案**：

```
CodeMirror 6 (纯文本编辑 + 语法高亮)
         ↓ onChange
remark.parse() 解析为 AST
         ↓
按 heading 分割为 Block 数组
         ↓
渲染可拖拽的 Block 列表 (用 dnd-kit 或 svelte-dnd-action)
拖拽结束后 → 重组 Markdown 文本 → 更新 CodeMirror
```

如果你想要 WYSIWYG 体验且愿意投入更多开发时间，Milkdown 是一个基于 ProseMirror 和 Remark 构建的插件驱动 WYSIWYG Markdown 编辑器，编辑器提供无缝的编辑体验，Markdown 语法实时渲染同时保持对文档结构的完全编程控制。 但对于你的场景，CodeMirror 6 + 自定义拖拽层更可控。

### 实时预览：Paged.js 替代纯 CSS 模拟

**关键改进：引入 Paged.js 作为分页引擎。**

Paged.js 是一个开源库，用于在浏览器中显示分页内容并使用 Web 技术生成印刷书籍。它包含一组 CSS 转换和分片布局的处理器，为 Paged Media 和 Generated Content CSS 模块提供 polyfill。

Paged.js 会在内容不适合一页时自动创建分页。但你也可以控制这种分片——通常在书中，你可能希望所有章节都从右页开始，有几个属性可以帮助你做到这一点。

具体架构：

```
Markdown 文本
    ↓ marked / remark
HTML 字符串
    ↓ 注入到预览容器
Paged.js Chunker 自动分页
    ↓
显示带分页的 A4 预览 (可设置 @page 尺寸、边距)
```

CSS 模板核心用 `@page` 规则：

```css
@page {
  size: A4;
  margin: 20mm 15mm;
  @bottom-center {
    content: counter(page);
  }
}

/* 手动分页：在 h1 前强制分页 */
h1 {
  break-before: page;
}

/* 避免在段落内部分页 */
p {
  orphans: 3;
  widows: 3;
}
```

@page 规则让你指定页面模型的各个方面，如尺寸、方向、背景、边距、裁切、注册标记等。所有影响页面布局的 CSS 属性都必须在其中声明。

### PDF 导出：Rust 后端生成（最可靠方案）

**放弃前端生成 PDF 的所有方案**，改用 Tauri 的 Rust 后端：

**推荐方案：Rust 侧调用 `headless Chrome/Chromium --print-to-pdf`**

```rust
// 在 Rust 侧使用 Command 调用
// 1. 将 Paged.js 渲染好的 HTML 写入临时文件
// 2. 调用系统 Chrome 或捆绑的 Chromium 进行 headless PDF 生成
std::process::Command::new("chrome")
    .args(["--headless", "--print-to-pdf=output.pdf", "file:///tmp/resume.html"])
    .output()
```

**备选方案：使用 Rust 的 `printpdf` crate**

使用 `printpdf` crate 在底层创建 PDF，目标是提供一个简单的 API，传入 `Vec<PdfBlock>` 来生成文档。但这需要自己实现 HTML→PDF 的布局逻辑，复杂度较高。

**最务实的方案：前端用 Paged.js 渲染 + 通过 Tauri 调用 `weasyprint` 或 `wkhtmltopdf`**

已有 Tauri V2 的打印插件支持 PDF/HTML 打印和打印机管理。 需要将 wkhtmltopdf 的二进制文件放在 `src-tauri/bin` 文件夹中，插件会自动在那里查找。

但 wkhtmltopdf 已停止维护，更推荐用 `weasyprint`（Python）或直接捆绑 Chromium headless。

### 字体策略修正

放弃"全量嵌入字体"的思路，改为：

1. **优先使用系统字体栈**：`font-family: "PingFang SC", "Microsoft YaHei", "Noto Sans CJK SC", sans-serif;`
2. **可选嵌入**：只嵌入一个轻量字体子集（用 `fonttools` 的 `pyftsubset` 工具），将 16MB 压缩到 **1-2MB**
3. **PDF 导出时**：由 headless Chrome 处理字体嵌入，它会自动将使用到的字体子集嵌入 PDF

## 六、关键风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Paged.js 实时 re-render 性能 | 加 debounce (300ms)，仅在用户停止输入后重新分页 |
| headless Chrome 增大安装包 | 不捆绑 Chrome，使用用户系统已安装的浏览器，通过 `which chromium` / 注册表探测路径 |
| 跨平台字体差异 | 在 PDF 导出时使用 `--font-render-hinting=none` 参数保证一致性 |
| Paged.js 与实际打印结果有偏差 | Paged.js 是 W3C 分页媒体规范的 polyfill，但 Paged.js 和 Chrome 在如何解释 W3C 分页媒体规范的几个重要方面存在分歧——因此最终以 headless Chrome 输出为准，Paged.js 仅作预览 |

---

## 七、总结

原方案的主要问题：
1. **`window.print()` 在 Tauri 中不可靠** — 这是最大隐患
2. **bytemd 维护停滞且不支持块拖拽** — 核心需求无法满足
3. **纯 CSS 模拟没有真正分页** — 预览失真
4. **字体嵌入与轻量目标矛盾** — 包体积超标

修订方案的核心改动：`CodeMirror 6` + `Paged.js` + `Rust 侧 headless Chrome PDF 导出` = 可靠、可控、轻量。
